## 1. File structure

Generator files are plain UTF-8 text. Sections are separated by blank lines; blank lines inside a table body end the current item.

```
[Header commands]        ‚Üê optional, before any Table:

Table: FirstTable        ‚Üê entry point (first table in file)
...items...

Table: SecondTable
...items...
```

The **first table** in the file is always the entry point. When the generator runs, it picks a random item from that table and expands it.

---

## 2. Header commands

Placed **before any `Table:` line**. All are optional.

### `Header:`
```
Header: <b>My Generator</b>
```
Text (or HTML) displayed above the generated output. HTML tags are rendered in the GUI.

### `Footer:`
```
Footer: Generated by Sparkwyrd
```
Text (or HTML) displayed below the generated output.

### `MaxReps:`
```
MaxReps: 5
```
Maximum number of repetitions that can be run. If the user requests more, the count is capped.

### `Formatting:`
```
Formatting: html
Formatting: text
```
Controls how formatting filters (`bold`, `italic`, `underline`) render. Default: `html`. In `text` mode, bold produces upper-case, italic wraps in `*`, underline in `"`.

### `Title:`
```
Title: My Page Title
```
Sets the HTML `<title>` when used via CGI or CLI with HTML output.

### `Use:`
```
Use: names/HumanNames.ipt
```
Includes tables from another `.ipt` file. Included tables are available by name in the current file. *(Not yet implemented in this build.)*

### `Prompt:`
```
Prompt: Character Name {} Aldric
Prompt: Class {Fighter|Wizard|Rogue|Cleric} Fighter
```
Adds user-input fields. Empty `{}` = free-text entry. `{opt1|opt2|...}` = drop-down list. Values are accessible as `{$prompt1}`, `{$prompt2}`, etc. *(Not yet implemented in this build.)*

### `Set:` (global)
```
Set: world = [@WorldNames]
```
Evaluates the expression **once** when the generator runs and stores the result. Available as `{world}` in any table.

### `Define:` (global constant)
```
Define: tension = {1d6}
```
Stores the expression **unevaluated**. Each time `{tension}` is referenced, `{1d6}` is re-rolled. *(Not yet implemented in this build.)*

## 3. Table definition

```
Table: TableName
```

Starts a new table called `TableName`. Everything below (until the next `Table:` or end of file) is part of this table.

Table names are **case-sensitive**. Spaces are allowed in names.

### Table-level directives

Placed inside a table body, before (or mixed with) items:

```
Type: Weighted          ‚Üê default; items have numeric weights
Type: Lookup            ‚Üê items have range keys (1-5:, 6-8:, etc.)
Type: Dictionary        ‚Üê items have text keys
Roll: 1d10              ‚Üê dice expression for Lookup tables
Default: some text      ‚Üê result when no item matches (Lookup/Dictionary)
Shuffle: OtherTable     ‚Üê reset OtherTable's deck before this table runs
EndTable:               ‚Üê explicitly ends the table (rarely needed)
```

### `Set:` inside a table

```
Table: HeroNames
Set: gender = male
Aldric
Sylvara
```

`Set:` lines inside a table are **directives**, not items. They always run before any item is picked. Use them to set variables that items in this table will share.

---

## 4. Table items

Each non-blank, non-directive line is a table item.

### Default weight (1)
```
Table: Colors
red
green
blue
```
Each color has equal probability.

### Explicit weight
```
Table: Colors
5:red
2:green
blue
```
`red` appears 5x more than `blue` (which has default weight 1). `green` appears 2x more than `blue`.

### Line continuation (`&`)
Items must be on a single line. Use `&` at the end of a line to continue on the next:
```
Table: LongParagraph
This is a very long result that &
spans multiple lines in the source &
but appears as one line in output.
```

---

## 5. Table types

### Weighted (default)
Items are picked randomly according to their numeric weights. No `Type:` command needed.

### Lookup
```
Table: EncounterTable
Type: Lookup
Roll: 1d12
Default: Nothing happens
1-2:A lone scout
3-5:A patrol of three
6-8:A full squad
9-10:An elite unit
11-12:The commander
```
`Roll:` determines which die is rolled. Each item has a range `min-max:` or a single value `n:`. If the roll falls outside all defined ranges, `Default:` is used.

Call with `[@EncounterTable]` for a random roll, or `[#n EncounterTable]` to pick item at index n.

### Dictionary
```
Table: HitDice
Type: Dictionary
Default: d6
fighter: d10
wizard: d4
cleric: d8
rogue: d6
```
Items have text keys instead of numeric weights. Access with `[#{$class} HitDice]` where `{$class}` holds a key like "fighter".

## 6. Special characters

These sequences are processed in the **final output** (not in the source file).

| Sequence | Output |
|---|---|
| `\n` | Newline |
| `\t` | Tab character |
| `\z` | Empty string ‚Äî makes a blank item explicit (not ignored by the parser) |
| `\_` | Forced space ‚Äî useful when a space would otherwise be trimmed |
| `\a` | Inserts "a" or "an" depending on the next letter (English heuristic) |
| `\` | Escape: `\{3d6}` outputs the literal text `{3d6}` instead of rolling dice |

### Example
```
Table: Creatures
\a ancient [@CreatureType]
```
Outputs `an ancient dragon` or `a goblin` etc.

---

## 7. Comments

Three styles of comment are supported:

```
# This entire line is a comment (hash at start)
; This entire line is a comment (semicolon at start)
// This entire line is a comment (double-slash at start)

This is a table item // but everything after the double-slash is ignored
```

## 8. Expressions

Expressions are placed between curly braces `{...}`. They are evaluated and replaced with their result.

### Dice rolls
```
{1d6}         ‚Üí roll 1d6 (result: 1‚Äì6)
{2d10+3}      ‚Üí roll 2d10, add 3
{1d6*10}      ‚Üí roll 1d6, multiply by 10
{3d6-2}       ‚Üí roll 3d6, subtract 2
```

### Math expressions
```
{1+2*3}       ‚Üí 7
{(4+6)*2}     ‚Üí 20
{1d6 * 10 + 2d4}
```

Supported operators: `+` `-` `*` `/` `^` (power)

### Math functions
```
{max(1d6, 1d6)}     ‚Üí higher of two d6 rolls
{min(1d8, 1d8)}
{round(1d20 / 6)}
{abs(-3)}
{floor(7.9)}         ‚Üí 7
{ceil(7.1)}          ‚Üí 8
{sqrt(16)}           ‚Üí 4.0
```

### Text in expressions
```
{'hello' + 'world'}       ‚Üí helloworld
{if(1d20 > 15, 'hit', 'miss')}
{substr('Sparkwyrd', 1, 5)}   ‚Üí Spark
{length('hello')}              ‚Üí 5
{trim('  spaces  ')}
```

Conditional function: `if(condition, value_if_true, value_if_false)`

## 9. Variables

### Setting a variable

**Inside a table (directive form):**
```
Table: MyTable
Set: hero = [@HeroNames]
The hero {hero} went to battle.
```
`Set:` directives always run before any item is picked. `{hero}` returns the same value throughout the table item.

**Inline assignment in an expression:**
```
{myroll=1d20}     ‚Üê evaluates, assigns to myroll, AND outputs the value
{myroll==1d20}    ‚Üê evaluates, assigns to myroll, does NOT output
```

**Inline assignment in a table call:**
```
[@hero=HeroNames]    ‚Üê rolls on HeroNames, assigns to hero, AND outputs
[@hero==HeroNames]   ‚Üê rolls on HeroNames, assigns to hero, does NOT output
```

### Reading a variable
```
{varname}      ‚Üê preferred form
{$varname}     ‚Üê legacy form (still supported)
```

### Variable scope

All variables are **global** within a single generator run. A variable set in one table is accessible in all other tables during the same run.

### Global variables (before any Table:)
```
Set: world = [@WorldNames]
```
Evaluated once when the generator starts. Available in all tables.

### Constants (Define:)
```
Define: tension = {1d6}
```
Unlike `Set:`, the expression is **not** evaluated at definition time. Each time `{tension}` is used, `{1d6}` is re-rolled fresh. *(Not yet implemented in this build.)*

## 10. Table calls

### Random roll `[@Table]`
```
[@Weapons]              ‚Üê one random result from Weapons table
[@3 Weapons]            ‚Üê three random results
[@{1d4} Weapons]        ‚Üê 1d4 random results (nested expression)
```

Multiple results are separated by a space. Use `>> implode` to join with commas.

### Pick by index `[#N Table]`
```
[#5 MyTable]            ‚Üê returns item number 5 (1-based)
[#{level} DifficultyTable]  ‚Üê index from variable
```

For Dictionary tables, the index is a text key:
```
[#{$class} HitDice]
```

### Deck pick `[!N Table]`
```
[!Skills]               ‚Üê one draw from Skills (without replacement)
[!4 Skills]             ‚Üê four draws, no duplicates
```

Once all items have been drawn, the deck is empty. Use `Shuffle:` to reset it.

### Dynamic table names (nesting)
```
[@{theme}_table]        ‚Üê if theme="fantasy", calls [@fantasy_table]
[@[@GetTableName]]      ‚Üê picks the table name from another table
```

## 11. Inline choices

```
[|option1|option2|option3]
```

Picks one option at random with equal probability. Options cannot be weighted. Options can be nested and can contain any other expression.

```
[|[@RareItem]|[@CommonItem]|nothing]
```

## 12. Filters

Filters transform the result of a table call or literal text. Apply with `>>`:

```
[@Monsters >> upper]
[@Names >> proper]
[!4 Skills >> sort >> implode]
[literal text >> bold]
```

Multiple filters can be chained: the output of each filter feeds into the next.

### Case filters
| Filter | Effect |
|---|---|
| `lower` | All lowercase |
| `upper` | ALL UPPERCASE |
| `proper` | First Letter Of Each Word Capitalised |

### Formatting filters (HTML-aware)
| Filter | HTML output | Plain text |
|---|---|---|
| `bold` | `<b>text</b>` | `TEXT` (uppercase) |
| `italic` | `<i>text</i>` | `*text*` |
| `underline` | `<u>text</u>` | `"text"` |

### String filters
| Filter | Effect |
|---|---|
| `trim` | Remove leading and trailing whitespace |
| `ltrim` | Remove leading whitespace |
| `rtrim` | Remove trailing whitespace |
| `length` | Return the character count |
| `reverse` | Reverse the string character by character |
| `left N` | Return first N characters (default 1) |
| `right N` | Return last N characters (default 1) |
| `substr S L` | Substring starting at position S, length L (0 = to end) |
| `at sub` | Position of substring `sub` (0 if not found) |
| `replace /find/repl/` | Replace all occurrences of `find` with `repl` |
| `+- / plusminus` | Format a number with explicit sign: `+5`, `-3`, `+0` |

### List filters (for multiple-result calls)
| Filter | Effect |
|---|---|
| `sort` | Sort results alphabetically |
| `implode` | Join results with `, ` (default separator) |
| `implode <sep>` | Join with a custom separator string |

### Processing filters
| Filter | Effect |
|---|---|
| `each TableName` | Pass each result to TableName as parameter `{$1}`; replace with output |
| `eachchar TableName` | Pass each character to TableName as `{$1}`; replace character with output |

## 13. Conditionals

### Basic form
```
[when]condition[do]expression[end]
```

### With else
```
[when]condition[do]expression_if_true[else]expression_if_false[end]
```

### Negated form
```
[when not]condition[do]expression[end]
[when not]condition[do]expression[else]expression_if_true[end]
```

### Condition forms

| Form | True when |
|---|---|
| `{$x} = value` | Variable equals value (numeric or alphabetic) |
| `{$x} > 5` | Numeric greater-than |
| `{$x} < 5` | Numeric less-than |
| `{$x} >= 5` | Greater-than-or-equal |
| `{$x} <= 5` | Less-than-or-equal |
| `{$x} <> value` | Not equal |
| `{$x}` (no operator) | True if `{$x}` is non-empty |

If both sides of a comparison are numeric, numeric comparison is used; otherwise alphabetical.

> **Note:** Conditionals cannot be nested.

### Examples
```
[when]{roll} > 3[do]Success![else]Failure.[end]

[when not]{$class}[do][class==warrior][end]

[when]{$race} = elf[do][@ElfSpells][else][@HumanSpells][end]
```

## 14. Deck picks and Shuffle

Deck picks draw items from a table **without replacement**, like drawing cards from a deck. The deck starts full at the beginning of each generator run.

```
Table: QuestTable
Shuffle: Skills          ‚Üê reset Skills deck when QuestTable is called
[@hero]'s skills: [!5 Skills >> sort >> implode]
```

`Shuffle:` inside a table resets the named table's deck so all items are available again before drawing.

```
Table: Skills
Navigation
Swordplay
Gunnery
Seamanship
Lockpicking
Swimming
```

Each call to `[!Skills]` removes the selected item; `[!5 Skills]` draws 5 without repeats. When the deck is empty, no more items are returned.

## 15. Parameters (with)

Pass values into a sub-table call:

```
[@stat_block with Strength, {3d6}]
```

Inside the called table, parameters are accessed as `{$1}`, `{$2}`, etc.:

```
Table: stat_block
{$1}: {$2}  (modifier: [{$2} - 10 >> +-])
```

Any expression can be passed as a parameter: literal text, dice rolls, variables, or even other table calls.

## 16. Built-in variables

Automatically set at the start of each generator run:

| Variable | Value |
|---|---|
| `{app}` | Name of the program (`Sparkwyrd`) |
| `{version}` | Engine version |
| `{os}` | Operating system |
| `{date}` | Current date (localised format) |
| `{time}` | Current time (HH:MM) |
| `{rep}` | Current repetition number (starts at 1) |
| `{formatting}` | `html` or `text` |
| `{cli}` | `yes` if running from CLI, empty otherwise |
| `{fullpath}` | Full path of the generator file *(pending)* |
| `{docpath}` | Directory of the generator file *(pending)* |
| `{self}` | Filename only *(pending)* |

## 17. HTML output

When `Formatting: html` (the default), certain text patterns are rendered with HTML in the GUI:

- The `Header:` and `Footer:` values support HTML.
- The `bold`, `italic`, and `underline` filters produce `<b>`, `<i>`, `<u>` tags.
- Generator output can include HTML tags directly ‚Äî they will be rendered in the GUI and stripped in `--plain` CLI mode.

**Supported tags in output:**

| Tag | Effect |
|---|---|
| `<b>`, `<strong>` | **Bold** |
| `<i>`, `<em>` | *Italic* |
| `<u>` | Underline |
| `<h1>` `<h2>` `<h3>` | Headings (large, medium, small) |
| `<code>`, `<tt>` | Monospace, shaded background |
| `<br>`, `<br/>` | Line break |
| `<p>`, `</p>` | Paragraph break |
| `<hr>` | Horizontal rule |

**HTML entities:** `&amp;` `&lt;` `&gt;` `&quot;` `&apos;` `&nbsp;`

## Appendix ‚Äî complete syntax quick reference

```
# Comments
# hash   ; semicolon   // double-slash (line-start or inline)

# Header section (before first Table:)
Header: <b>Title</b>
Footer: some text
MaxReps: N
Formatting: html | text
Title: page title
Set: var = expression
Define: const = expression
Use: path/to/file.ipt
Prompt: Label {opt1|opt2} default

# Table definition
Table: Name
Type: Weighted | Lookup | Dictionary
Roll: NdN
Default: fallback text
Shuffle: OtherTable
EndTable:

# Inside a table ‚Äî directives
Set: var = expression    ‚Üê always runs, not a random item

# Inside a table ‚Äî items
plain text               ‚Üê weight 1
3:weighted text          ‚Üê weight 3
1-5:range item           ‚Üê Lookup range
key:value                ‚Üê Dictionary entry
multi-line &             ‚Üê continues on next line

# Special output characters
\n  \t  \z  \_  \a  \{escaped}

# Expressions
{1d6}  {2d10+3}  {1d6*10}
{a+b}  {max(x,y)}  {if(c,t,f)}
{var}  {$var}
{x=expr}   {x==expr}  ‚Üê assign (+ output / silent)

# Table calls
[@Table]          [@N Table]        [#{N|key} Table]    [!N Table]
[@Table >> filter >> filter]
[@Table with p1, p2]
[@x=Table]   [@x==Table]

# Inline choice
[|a|b|c]

# Conditionals
[when]cond[do]expr[end]
[when]cond[do]expr[else]expr2[end]
[when not]cond[do]expr[end]

# Filters
lower  upper  proper
bold  italic  underline
sort  implode [sep]
trim  ltrim  rtrim  length  reverse
left N  right N  substr S L  at sub  replace /f/r/  +-

```
